NIMS Release Notes
release 0.1 2016-01-08

Features
  • Support for Kongsberg M3 Multi-beam sonar.
  • Real-time display of sonar data and EchoMetrics.
  • Tracks written to csv file.

Known Issues
The nims processes must be started and running before starting the web-based user interface, nims.py.  nims.py looks for the frame_buffer message queue which is created by the ingester module.  If the queue does not exist, nims.py runs but is not connected to the sonar device. 

Chrome is currently the only supported browser.

The tracks are not displayed in the user interface.

Tracking is not validated.

The user interface shows the current parameter settings from the config file, and the user can edit them, but there is no validation of the user input and the configuration does not actually get changed.  For new parameters to take effect, the nims processes need to be restarted and this should happen from the user interface.
